# Contributing to PolypPal

Thank you for your interest in contributing to PolypPal! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
- A clear, descriptive title
- Steps to reproduce the bug
- Expected behavior vs. actual behavior
- Screenshots if applicable
- Your Python version and operating system

### Suggesting Features

We love feature suggestions! Please open an issue with:
- A clear description of the feature
- Why it would be useful
- Example use cases
- Optional: mockups or examples

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow the existing code style
   - Add comments for complex logic
   - Test your changes thoroughly
4. **Commit with clear messages**
   ```bash
   git commit -m "Add feature: your feature description"
   ```
5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open a Pull Request**
   - Describe what your PR does
   - Reference any related issues
   - Include screenshots for UI changes

## Code Style

- Follow PEP 8 guidelines for Python code
- Use meaningful variable and function names
- Add docstrings to functions
- Keep functions focused on a single task

## Testing

Before submitting a PR:
- Test with different image sizes and formats
- Verify that measurements persist through rotation
- Check that Excel output is correct
- Test both manual mode and metadata mode

## Questions?

Feel free to open an issue with the "question" label, or reach out directly.
